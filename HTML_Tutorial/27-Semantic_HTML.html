HTML Semantic Elements

  • Semantic elements = elements with a meaning.
  • A semantic element clearly describes its meaning to both the browser and the developer.
  • Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.
  • Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content.
  • Some of the semantic Elements in HTML include
    • <article> specifies independent, self-contained content.
    • <aside> defines some content aside from the content it is placed in (like a sidebar).
    • <footer> defines a footer for a document or section and may contain
      • authorship information
      • copyright information
      • contact information
      • sitemap
      • back to top links
      • related documents
    • <header> container for introductory content or a set of navigational links.
    • <main> Specifies the main content of a document
    • <nav> defines a set of navigation links.
